**文件**: `src/common/interfaces.ts`

- `UserRole` - 用户角色枚举 (admin, project_manager, tester, viewer)
- `ProjectMember` - 项目成员

### 3. 认证模块 (auth) ✅
**目录**: `src/modules/auth/`

创建的文件:
- `auth.module.ts` - 认证模块定义
- `auth.service.ts` - 认证服务
  - 用户注册
  - 用户登录
  - Token 生成
  - 用户管理
  - 默认管理员创建
- `auth.controller.ts` - 认证控制器
  - POST /api/auth/login
  - POST /api/auth/register
  - GET /api/auth/profile
  - GET /api/auth/users
- `strategies/jwt.strategy.ts` - JWT 策略
- `guards/jwt-auth.guard.ts` - JWT 认证守卫
- `guards/permissions.guard.ts` - 权限守卫
- `decorators/public.decorator.ts` - 公开路由装饰器
- `decorators/permissions.decorator.ts` - 权限装饰器
- `index.ts` - 导出

### 4. 权限管理模块 (permissions) ✅
**目录**: `src/modules/permissions/`

创建的文件:
- `permissions.module.ts` - 权限模块定义
- `permissions.service.ts` - 权限服务
  - 添加项目成员
  - 移除项目成员
  - 更新成员权限
  - 检查用户权限
  - 获取用户项目列表
  - 获取项目成员列表
- `permissions.controller.ts` - 权限控制器
  - POST /api/projects/:id/members - 添加成员
  - DELETE /api/projects/:id/members/:userId - 移除成员
  - PUT /api/projects/:id/members/:userId/permissions - 更新权限
  - GET /api/projects/:id/members - 获取成员列表
  - GET /api/user/projects - 获取用户项目
- `index.ts` - 导出

## 🔧 配置说明

### JWT 配置

JWT Secret 可通过环境变量配置:
```bash
export JWT_SECRET=your-secret-key-change-this-in-production
```

默认值: `your-secret-key-change-this-in-production`

Token 有效期: 24 小时

### 默认管理员账号

首次运行时自动创建:
- 用户名: `admin`
- 密码: `admin123`
- 角色: `admin`

## 🚀 如何使用

### 1. 启动应用

```bash
cd d:\测试小工具\autodomp
npm start
```

### 2. 访问系统

打开浏览器: `http://localhost:3000`

系统会自动跳转到登录页面。

### 3. 登录

使用默认管理员账号:
- 用户名: `admin`
- 密码: `admin123`

1. 导航到 **项目** → **项目成员**
2. 选择项目
3. 添加成员并分配权限
