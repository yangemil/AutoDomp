# 用户角色和菜单权限管理系统设计文档

**日期**: 2026-03-01
**版本**: 1.0
**状态**: 已批准

## 1. 概述

为AutoDOMP系统添加完整的用户角色管理和菜单权限管理功能，包括：
- 用户管理页面（给用户授权角色）
- 角色菜单权限管理页面（给角色授权访问菜单）
- 动态菜单渲染（根据用户角色显示菜单）

## 2. 设计原则

- **基于角色的访问控制（RBAC）**: 采用固定角色模式（admin, project_manager, tester, viewer）
- **分层菜单结构**: 菜单按分组（如"测试管理"）和菜单项（如"测试用例"）组织
- **配置文件驱动**: 使用JSON配置文件存储角色菜单权限，无需修改代码
- **独立页面管理**: 用户管理和角色菜单权限管理作为独立页面
- **最小权限原则**: 默认为最小权限，按需分配

## 5. 后端API设计

### 5.1 用户管理API

#### 获取所有用户
- **方法**: `GET`
- **路径**: `/api/users`
- **权限**: admin
- **响应**:
```json
[
  {
    "id": "8eb51abf-901b-42c2-9804-bc80e5238a5b",
    "username": "admin",
    "email": "admin@autodomp.com",
    "role": "admin",
    "createdAt": "2026-02-28T14:59:14.347Z"
  }
]
```

#### 新增用户
- **方法**: `POST`
- **路径**: `/api/users`
- **权限**: admin
- **请求体**:
```json
{
  "username": "newuser",
  "email": "newuser@example.com",
  "password": "password123",
  "role": "tester"
}
```
- **响应**: 返回创建的用户信息（不含密码）

#### 更新用户角色
- **方法**: `PUT`
- **路径**: `/api/users/:userId/role`
- **权限**: admin
- **请求体**:
```json
{
  "role": "project_manager"
}
```
- **响应**: 返回更新后的用户信息

#### 删除用户
- **方法**: `DELETE`
- **路径**: `/api/users/:userId`
- **权限**: admin
- **响应**: 成功返回 `true`

### 5.2 角色菜单权限API

#### 获取所有角色菜单权限
- **方法**: `GET`
- **路径**: `/api/role-menu-permissions`
- **权限**: admin
- **响应**: 返回完整的角色菜单权限配置

#### 获取指定角色的菜单权限
- **方法**: `GET`
- **路径**: `/api/role-menu-permissions/:roleId`
- **权限**: admin
- **响应**: 返回指定角色的菜单权限

#### 更新角色菜单权限
- **方法**: `PUT`
- **路径**: `/api/role-menu-permissions/:roleId`
- **权限**: admin
- **请求体**:
```json
{
  "menus": [
    {
      "id": "home",
      "name": "首页",
      "path": "/"
    }
  ]
}
```
- **响应**: 返回更新后的角色菜单权限

#### 获取当前用户的菜单权限
- **方法**: `GET`
- **路径**: `/api/user/menu-permissions`
- **权限**: 已登录用户
- **响应**: 返回当前用户的菜单权限

## 6. 前端动态菜单渲染

### 6.1 渲染流程

```
用户登录
  ↓
获取token和用户信息
  ↓
调用 /api/user/menu-permissions
  ↓
根据权限配置动态渲染导航栏
  ↓
用户角色变更 → 重新加载菜单
```

### 6.2 实现方式

1. 在 `main.hbs` 中添加菜单渲染函数
2. 监听全局事件（如 `menuPermissionsUpdated`）
3. 动态更新导航栏DOM
4. 禁用/隐藏无权限的菜单项

### 6.3 菜单权限验证

- 前端路由守卫：访问菜单时验证权限
- 后端API权限守卫：所有API调用验证用户角色
- 403重定向：无权限访问时跳转到首页或提示

## 7. 安全性设计

### 7.1 认证与授权
- 所有API都需要JWT认证
- 用户管理和角色菜单权限仅admin可访问
- 角色菜单权限文件存储在服务端，前端无法直接修改

### 7.2 数据安全
- 密码加密存储（bcrypt）
- Token过期机制（24小时）
- 敏感操作需要二次确认（删除用户）

### 7.3 权限控制
- 角色菜单权限配置需要admin权限
- 用户无法修改自己的角色（防止权限提升）
- 删除用户前检查是否为当前登录用户

## 8. 技术栈

- **后端**: NestJS
- **前端**: HTML + CSS + JavaScript
- **数据存储**: JSON文件
- **认证**: JWT (jsonwebtoken)
- **密码加密**: bcrypt
