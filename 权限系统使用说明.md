# 权限系统使用说明

## 📋 概述

AutoDOMP 现已集成完整的 RBAC (基于角色的访问控制) 权限管理系统,支持用户认证、项目级权限控制和细粒度权限管理。

---

## 🔐 默认账号

### 管理员账号
- **用户名**: `admin`
- **密码**: `admin123`
- **权限**: 拥有所有项目的完全控制权限

> ⚠️ **重要**: 首次登录后请立即修改默认密码!

## 👥 用户角色

系统支持以下角色:

| 角色 | 权限级别 | 说明 |
|------|----------|------|
| **admin** | 最高权限 | 系统管理员,可以访问所有项目和管理用户 |
| **project_manager** | 高权限 | 项目管理员,可以管理分配的项目 |
| **tester** | 中等权限 | 测试人员,可以执行测试和查看报告 |
| **viewer** | 低权限 | 只读用户,只能查看项目信息 |

---

## 🔑 权限类型

每个项目成员可以拥有以下权限:

| 权限 | 说明 | 可执行操作 |
|------|------|------------|
| **read** | 只读 | 查看项目、测试用例、执行记录、报告 |
| **write** | 编辑 | 创建/编辑测试用例、修改项目配置 |
| **execute** | 执行 | 执行测试用例和场景 |
| **manage** | 管理 | 添加/移除项目成员、分配权限 |

## 📂 项目成员管理

### 添加项目成员

1. 登录系统
2. 导航到 **项目** → **项目成员**
3. 选择要管理的项目
4. 点击 **+ 添加成员** 按钮
5. 填写用户信息:
   - 用户名
   - 角色 (成员/管理员/访客)
   - 权限 (查看/编辑/执行/管理)
6. 点击 **添加**

### 编辑成员权限

1. 在项目成员列表中找到要修改的成员
2. 点击 **编辑** 按钮
3. 修改权限设置
4. 保存更改

### 移除项目成员

1. 在项目成员列表中找到要移除的成员
2. 点击 **移除** 按钮
3. 确认操作

## 👤 用户管理

### 管理系统用户

1. 登录系统（需要 admin 角色）
2. 导航到 **用户管理**
3. 查看所有用户列表
4. 操作：
   - **新增用户**: 点击 "+ 新增用户" 按钮，填写用户名、邮箱、密码和角色
   - **编辑角色**: 点击 "编辑角色" 按钮，选择新的角色
   - **删除用户**: 点击 "删除用户" 按钮，确认后删除

### 用户角色说明

| 角色 | 权限级别 | 说明 |
|------|----------|------|
| **admin** | 最高权限 | 系统管理员,可以访问所有功能和菜单 |
| **project_manager** | 高权限 | 项目管理员,可以管理分配的项目和菜单 |
| **tester** | 中等权限 | 测试人员,可以执行测试和查看报告 |
| **viewer** | 低权限 | 只读用户,只能查看项目信息和报告 |

---

## 🎨 角色菜单权限管理

### 管理角色菜单权限

1. 登录系统（需要 admin 角色）
2. 导航到 **角色菜单权限**
3. 展开要修改的角色
4. 勾选/取消勾选菜单项
5. 点击 "保存" 按钮应用更改

### 菜单权限说明

- **首页**: 系统首页
- **测试管理**: 测试用例、测试场景
- **定时**: 待执行、已执行
- **执行**: 执行记录、测试报告
- **日志**: 日志查询、日志管理
- **资源**: 步骤模板、选择器工具
- **项目**: 项目管理、项目成员
- **设置**: 系统设置
- **用户管理**: 管理系统用户（仅 admin）
- **角色菜单权限**: 管理角色菜单权限（仅 admin）

### 动态菜单

系统会根据用户的角色动态显示菜单：
- 用户只能看到其角色有权限的菜单项
- 角色菜单权限修改后，用户下次登录或刷新页面生效
- 访问无权限的页面时会被重定向到首页

## 🔧 API 认证

### 使用 Token 访问 API

所有需要认证的 API 都需要在请求头中携带 JWT Token:

```javascript
const token = localStorage.getItem('token');

fetch('/api/projects', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
```

### 公开 API (无需认证)

以下 API 不需要认证:

- `POST /api/auth/login` - 用户登录
- `POST /api/auth/register` - 用户注册

### 需要 Token 的 API

以下 API 需要认证:

- `GET /api/projects` - 获取项目列表
- `POST /api/projects` - 创建项目
- `GET /api/test-cases` - 获取测试用例
- `POST /api/test-cases` - 创建测试用例
- `POST /api/execute/:id` - 执行测试
- 等等...

## 📝 注册新用户

### 方法 1: 通过登录页面

1. 访问 `http://localhost:3000/login`
2. 点击 **没有账号? 立即注册**
3. 填写注册信息:
   - 用户名
   - 邮箱
   - 密码 (至少6位)
   - 确认密码
4. 点击 **注册**

### 方法 2: 通过 API

```bash
curl -X POST http://localhost:3000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "testuser",
    "email": "test@example.com",
    "password": "password123"
  }'
```

## 🔒 安全建议

### 1. 修改默认管理员密码

首次登录后立即修改默认密码:

```javascript
// 通过 API 修改密码 (需要实现)
fetch('/api/auth/change-password', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    oldPassword: 'admin123',
    newPassword: 'your-strong-password'
  })
});
```

### 2. 使用强密码

密码要求:
- 至少 8 个字符
- 包含大小写字母
- 包含数字
- 包含特殊字符

### 3. 定期更新 Token

JWT Token 默认有效期为 24 小时,过期后需要重新登录。

### 4. 最小化权限分配

遵循最小权限原则:
- 只分配用户工作所需的权限
- 定期审查和调整权限
- 及时移除离职用户的访问权限

## 🐛 常见问题

### Q1: 登录后自动跳转到登录页面?

**原因**: Token 无效或已过期

**解决方案**:
1. 清除浏览器 localStorage
2. 重新登录

### Q2: 无法访问某些项目?

**原因**: 没有该项目的访问权限

**解决方案**:
1. 联系项目管理员
2. 请求添加为项目成员

### Q3: 无法执行测试?

**原因**: 没有执行权限

**解决方案**:
1. 检查项目权限设置
2. 联系项目管理员分配执行权限

### Q4: 如何重置密码?

**当前版本**: 暂不支持密码重置,请联系系统管理员

**未来版本**: 将支持邮件重置密码功能
